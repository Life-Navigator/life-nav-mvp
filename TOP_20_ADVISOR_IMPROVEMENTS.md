# Top 20 Advisor Improvements (Phase 8)

Ranked from the **40 live advisor turns** on v89 (12 personas + adversarial + 16 hard-decision probes).
Every item is a _quality_ finding — **trust/safety is clean** (0 fabrication, 0 advice, 0 hallucination;
see `BETA_GO_NO_GO_REPORT.md` Phase 7). Severity: **S1** = blocks good beta UX · **S2** = noticeable ·
**S3** = polish.

## The dominant pattern

The advisor is **safe but evasive on decisions**. Asked a concrete decision question, it reflects the
question back and asks an open _vision/motivation_ question instead of engaging the specifics the user just
provided. Measured: **9/16** hard-decision turns ignored the financial context the user gave one turn
earlier ("$120k income, $60k saved, $450k house" → "Can I afford this?" → _"how much cash would you want
left after the down payment?"_ with no reference to the numbers).

## Summary

- **Top 3 to move the beta needle:** #1 (use stated context), #2 (decision-framing instead of vision
  deflection), #6 (latency/streaming). These three convert "safe but evasive and slow" into "safe,
  responsive, and useful."
- **#10–11** are operational must-dos (apply the migration) so the beta is observable.
- Everything here is **quality**, not trust — the safety foundation held across all 40 live turns.
