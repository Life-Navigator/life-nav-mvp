# Dashboard Canonical Data Audit

_Sprint: Dashboard Trust Fix — No Invented Risks + Canonical Domain Summary Alignment (2026-06-13)._

The dashboard must show only real saved data, real computed summaries, real recommendation-engine output,
and honest missing-data states. This audit records every dashboard surface, its canonical source, the
mismatch found, and the fix.

## Part A — Risks / Opportunities grounding

| Surface                                                                           | Was                                                                                                                                                                                                                                                                                                                                   | Now                                                                                                                                                                                                                                                                                                                                                                                     |
| --------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

Archetype-template labels (`GENERIC_RISK_OPP_LABELS`, derived from `ROOT_OBJECTIVES`) are filtered out. Genuinely user-specific (non-archetype) snapshot items are kept. Empty → honest empty state. The Life Graph still renders the templates as relationship nodes (untouched). |
| `highest_priority_issue` (`next_best_action`)                                     | Only rendered if a ranked rec existed; otherwise hidden.                                                                                                                                                                                                                                                                              | If a grounded ACTION/OPPORTUNITY exists → "next best action". ## Part D — Missing-migration awareness (family_members / pets)

The `family.family_members` / `family.pets` / `family.guardianship` migration (`20260613000000`) is committed
but not yet applied to the live DB. The Family card distinguishes a **failed fetch (table missing)** from an
empty list: members/pets counts are `null` ("—") and the card shows "Family members and pets aren't enabled
yet (pending a database update). Your dependents, beneficiaries, contacts, and advisors are shown above." It
never reports `0` as if the user has no family. Once the migration is applied, the counts populate and RLS
isolates per user.

## Remaining limitations

- Education has no Core canonical summary endpoint; its card uses course counts + discovery coverage.
- Health remains intentionally gated (beta plan); the card does not inflate readiness.
- members/pets/guardianship counts require the `20260613000000` migration to be applied (`supabase db push`).
- Grounded risks/opportunities depend on the recommendation engine having produced evidence-backed
  RISK/OPPORTUNITY recs; with limited data the dashboard correctly shows the honest empty/"forming" states.
