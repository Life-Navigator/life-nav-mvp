## What the trace proves

1. **Native text extraction works** (PyMuPDF) for native-text PDFs; OCR is genuinely absent (scanned → `needs_review`).
2. **Extraction is deterministic labeled-field parsing** — real wills/trusts/policies (prose or scanned) yield few/zero fields. (Quality ceiling, but see below — not the dominant break.)
3. **The extracted facts terminate in the `documents` schema.** `documents.py` has **zero** writes to the life model (`life.*`) and never calls the MCP `IngestionService`. Verified by grep.
4. **No consumer reads `document_fields` values.** FamilyService reads `estate_plan`/`guardianship_plan`/`insurance_profiles` (CRUD tables), not `document_fields`. recommendations_os + my_life read only document **existence**. So an uploaded policy's `coverage_amount`, a will's `executor`, a trust's `trustee` are stored and then **never surfaced anywhere**.

## Consequence (the trust break)

Upload → store → extract → `document_fields`. **Stop.** The chain to Life Model / Family Readiness / Dashboard / Report / Recommendation **values** does not exist. Hence "Upload successful" with no visible downstream value — exactly the reported symptom. See `ROOT_CAUSE_ANALYSIS.md`.
