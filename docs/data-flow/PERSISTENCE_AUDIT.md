## Answers to the audit questions

### NOT persisted (computed-on-read or absent)

- **emotional_signals** — computed by `emotional_signals()` (life_discovery.py:254) on every snapshot; never stored.
- **dominant_narrative / narrative_explanation / goal_portfolio** — derived on read in `snapshot()` (life_discovery.py:868).
- **`life.motivations` rows** — table EXISTS (migration 154:40, cols `id,user_id,tenant_id,objective_id,text,kind,created_at`) but **no code path writes it**. The motivational signal is captured as `emotional_signals` instead.
- **structured wedding/goal dates** — never parsed; only free text.
- **multiple constraints from the opening statement** — only the _answered_ constraint step writes a row (#8) + any conflict/affordability constraints from `_detect_constraints` (#11). "money is tight + energy drained" as two separate constraints are NOT each written unless given as the constraint answer.
- **`life.risks` from discovery** — intentionally NOT written (life_discovery.py:829 TRUST RULE; risks come from the Recommendation OS).

### Persisted but NOT canonical (lived in a sub-store, not the aggregate read-path)

- candidate_goals were persisted (#4) and exposed in `snapshot.goal_portfolio`, but `my_life()` did not surface `goal_portfolio` until this sprint.
- `narrative_summary` / `time_horizon` lived in `life_vision.prompts` and reached `snapshot` only partially; `my_life()` never exposed `narrative_summary` or `timeline`.

### Persisted but NOT exposed (on /v1/life/my-life specifically — fixed this sprint)

- `dominant_narrative`, `narrative_summary`, `goal_portfolio`, `emotional_signals`, `timeline`, `coverage`, `missing_context` — all present in snapshot/health but absent from the my-life aggregate. **Now surfaced** (see MISSING_PERSISTENCE_FIX_REPORT).
