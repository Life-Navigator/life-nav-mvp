Sprint: **Data Flow & Rendering Integrity**. Scope: core-api discovery (LLM-free) → persistence → snapshot/API.
All line refs are `app/services/...` unless noted. Discovery is `RelationshipManager` + `LifeDiscoveryService` — **no LLM**.

## The traced onboarding statement

> "I'm getting married next June, then we want to buy a home and start a family. I'm gunning for a
> promotion to director, I'd like to do a Masters in AI, and I want to get in better shape. Financially
> I'm doing fine — comp is good — but money is tight month to month and my energy is drained from overwork."

This is fed as the `primary_goal` answer (and constraint/free turns). Trace item-by-item.

### Detection + classification path

`RelationshipManager.answer()` (relationship_manager.py:264) for a `kind=="goal"` step:

- calls `LifeDiscoveryService.discover_goal()` (life_discovery.py:778) → ONE root objective + graph,
- calls `analyze_statement()` (life_discovery.py:703) → **clause split** (`re.split` on `.`/`,`/`and`/`then`/… line 712) → per-clause `analyze()` (line 600) → `candidate_goals[]`,
- `converse()` (relationship_manager.py:410) re-extracts from the raw message and persists via `_persist_candidate_goals()` (relationship_manager.py:108).

`_goal_domain()` (life_discovery.py:160) tags each clause by keyword evidence; never invents `career` without explicit job terms.

### Honest free-text vs structured vs absent

- **Structured**: candidate goals (domain/status/confidence/quote), the primary objective + graph, risk profile, the answered constraint.
- **Free text only**: the wedding date ("next June"), `time_horizon`, the narrative text. **No date is ever parsed** (deliberate — see residual gaps).
- **Computed-on-read, not persisted as rows**: emotional_signals, dominant_narrative, narrative_explanation, goal_portfolio.
- **Absent (never written)**: `life.motivations` rows (table exists, migration 154 line 40); multiple constraints from the opening statement (only the _answered_ constraint step writes one row); `life.risks` from discovery (intentional TRUST RULE — risks come from the Recommendation OS, life_discovery.py:829).
