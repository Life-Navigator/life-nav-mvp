## 8. Migrations & background jobs

- **SQL migrations in core-api: NONE** (`git ls-tree … | grep -iE 'migration|\.sql'` → empty). Schema lives in `/supabase` migrations elsewhere in the monorepo; several advisor features reference tables (`candidate_goals`, `analytics.model_usage`) that must already exist or be created outside this service. `USAGE_TRACKING_ENABLED` comment notes it "needs migration."
- **Background jobs: NONE** inside this service — it is a synchronous orchestration tier. Fly config: `min_machines_running=1`, `auto_stop_machines="suspend"` (`fly.toml`). Streaming `/discovery/chat/stream` is request-scoped SSE, not a job.
