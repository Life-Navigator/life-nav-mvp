# Constraint Discovery (Phase 5)

**Date:** 2026-06-16 · Constraints are captured before objectives drive anything.

## Sources

- **Stated constraints:** the FLOW `constraint` step ("What feels like the biggest thing standing in your way?") persists to `life.constraints` and is exposed in `snapshot().active_constraints`.
- **Signal-derived constraints:** `emotional_signals()` flags money_stress (financial constraint), burnout (time/energy constraint), money_fine (finance NOT a constraint) — these shape the dominant narrative (e.g. distress ⇒ stabilization-first; money_fine ⇒ balance, not wealth).

## Use

Constraints feed the dominant narrative ordering (stabilize-before-optimize) and the tradeoff question. The objective layer is generated only after constraints are understood (Phase 8).

## Honest scope

Constraint capture combines the existing structured step with the emotional-signal layer (deterministic keywords). It does not yet quantify constraint severity (e.g. a debt-to-income figure) — that requires real financial data, available post-discovery via the finance domain.
