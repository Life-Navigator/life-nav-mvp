CRITICAL FINDINGS: 5
================================================================================

1. CREDENTIALS EXPOSED IN REPOSITORY
   - backend/.env contains real JWT secret and database credentials
   - services/api/.env contains real credentials
   - Git history exposes credentials even if deleted
   ACTION: Immediately rotate all exposed secrets and purge git history

2. MISSING CONTAINER IMAGE SCANNING
   - No Trivy or equivalent vulnerability scanning in CI/CD
   - CVE detection not enforced pre-deployment
   ACTION: Add container image scanning to GitHub Actions pipeline

3. INSECURE TERRAFORM STATE
   - Terraform state bucket (GCS) not encrypted
   - No versioning or access restrictions configured
   - Bucket name exposed in source code
   ACTION: Enable versioning, encryption (CMEK), and restrict IAM access

4. NON-ROOT USER NOT ENFORCED
   - services/api/Dockerfile runs as root
   - services/qdrant/Dockerfile runs as root
   ACTION: Add non-root user creation and USER directive

5. HARDCODED DEVELOPMENT CREDENTIALS
   - docker-compose.yml contains 10+ hardcoded passwords
   - These teach bad security practices even for dev environment
   ACTION: Use .env files and environment variable substitution

================================================================================
HIGH PRIORITY FINDINGS: 8
================================================================================

6. Floating Docker Base Image Tags
   - Multiple Dockerfiles don't pin image digests
   - python:3.12-slim, node:20-alpine can pull unexpected updates
   - Recommendation: Use full SHA256 digests

7. Internal Firewall Rules Too Permissive
   - Allows all TCP ports (0-65535) internally
   - Should restrict to: 5432, 6379, 7687, 443, 50051
   - Impact: Lateral movement risk

8. Network Policy Overly Permissive
   - Database access uses wildcard namespace selectors
   - Should specify exact namespaces with labels
   - Impact: Cross-namespace access possible

9. Secret Refresh Interval Too Long
   - External Secrets Operator refresh = 1 hour
   - Recommendation: 15 minutes for production
   - Impact: 60-minute delay in secret rotation

10. Image Pull Policy Not Enforced
    - imagePullPolicy: IfNotPresent
    - Old cached images could be used
    - Recommendation: Always pull with digest verification

11. CI/CD Dependency Checks Not Blocking
    - Audit failures non-blocking on feature branches
    - CVSS threshold set to 7 (should be 5)
    - Impact: Vulnerabilities can merge

12. No Secret Rotation Automation
    - Secrets never automatically rotate
    - No Cloud Scheduler jobs defined
    - Impact: Credential compromise extends indefinitely

13. CORS Configuration Overly Permissive
    - ConfigMap has CORS_ORIGINS: ["*"]
    - Ingress has hardcoded allowed origins
    - Impact: Any domain can access API
